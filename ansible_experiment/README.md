# Ansible Experiment

This is an experiment to learn ansible. All the Ansible code is written by Antigravity, Google's AI coding assistant.

## Configuration of VMs:

I have used 3 VMs in Virtual Box to automate using Ansible. The VMs used had the following configurations:
- Ubuntu-server 24.04
- 2GB RAM, 1 CPU
- NAT Network Adapter, with consecutive static IP addresses for ease of access.
- Python3.12 installed in all of them by default.
- SSH Enabled

My host machine was used as control node for Ansible

## Goal

The goal is to use ansible to automate the following tasks:

1. Install Git
2. Install Nginx
3. Clone a git repo
4. Install dependencies
5. Run a python server
6. Configure nginx as a reverse proxy
7. Shutdown all servers

## Ansible Playbooks

The following ansible playbooks are used:

- `install_git.yaml`: Installs git
- `install_nginx.yaml`: Installs nginx
- `git_load.yaml`: Clones a git repo, installs dependencies, and runs a python server, and configures nginx as a reverse proxy
- `shutdown_all.yaml`: Shuts down all servers

Ansible configuration and inventory are present in `ansible.cfg` and `nginx_inventory.ini` respectively.

## Environment used:
- Debian 13.4 
- Ansible Core 2.19.4
- Python 3.13.5
- Antigravity IDE (all code generated by Antigravity)

## Running Ansible Playbooks

The ansible playbooks can be run using the following command:

```bash
ansible-playbook <playbook_name>
```

## Steps to run playbooks:
1. `ansible-playbook install_git.yaml`
2. `ansible-playbook install_nginx.yaml`
3. `ansible-playbook git_load.yaml`
4. `ansible-playbook shutdown_all.yaml`
